# 非天然氨基酸文献获取 — 架构说明

## 1. 总体架构

```
nnaa_fetch.py（入口）
    │
    ├─ search ──► lib/fetch_literature.py
    │                 ├─ nnaa_keywords / nnaa_queries（检索式）
    │                 ├─ pubmed_fetcher / crossref_fetcher
    │                 └─ deduplicator / filters
    │
    ├─ ai ──────► lib/ai_screener.py
    │
    └─ export ──► export_key_results.py → output/key_results/关键_*.csv
```

## 2. 目录职责

| 路径 | 职责 |
|------|------|
| **根目录 `nnaa_*.py`** | NNAA 专用：入口、关键词、查询、关键输出 |
| **`lib/`** | 通用管线：抓取、去重、AI、数据源 API |
| **`scripts/`** | 可选辅助：AI 配置检查、试跑、CSV 修复 |
| **`docs/`** | 使用说明 + 架构说明 |
| **`output/`** | 原始 CSV + `key_results/` 关键交付 |
| **`state/`** | `pending_ai.json`、`ai_checkpoint.json` |
| **`logs/`** | 运行日志 |

## 3. 数据流

```
PubMed + CrossRef（五轨道）
        ↓
去重 / 类型过滤 / 清洗
        ↓
state/pending_ai.json
        ↓
AI 批处理（ai_pass、ai_nnaa_category …）
        ↓
output/YYYY-MM-DD/*.csv
        ↓
export_key_results → 关键_代谢通路.csv 等
```

## 4. 五轨道

| 轨道 | 主题 |
|------|------|
| pathway | 代谢通路 |
| enzymatic | 酶法合成 |
| fermentation | 生物发酵 |
| chemical | 化学合成 |
| hybrid | 酶-化学联用 |

关键词定义见 `nnaa_keywords.py`。

## 5. 扩展

- 改关键词：编辑 `nnaa_keywords.py`
- 改 AI 标准：编辑 `config.yaml` → `ai_screening`
- 改关键输出规则：编辑 `export_key_results.py`

## 7. 检索噪声治理记录（2026-07-20）

抽样分析发现以下噪声来源及修复：

### 7.1 Hybrid 轨道严重噪声（最高优先级）

**问题**：`_HYB_SIG` 包含 `"hybrid synthesis"` 使 PubMed 抓入航空涡桨等无关论文；
CrossRef `_CR_HYB_METHODS` 中 `"hybrid catalysis"` / `"one-pot synthesis"` 过于通用。

**修复**：
- PubMed `_HYB_SIG`：移除 `"hybrid synthesis"`，保留 `"chemoenzymatic"`、`"sequential biocatalysis"`、`"one-pot biocatalysis"`、`"enzyme-chemical cascade"`
- CrossRef `_CR_HYB_METHODS`：移除 `"hybrid catalysis"`、`"one-pot synthesis"`，替换为 `"chemoenzymatic preparation"`、`"chemoenzymatic route"`、`"one-pot biocatalysis"`
- CrossRef hybrid extras：统一添加氨基酸语境约束（`"one-pot chemoenzymatic amino acid"` 等）

### 7.2 Pathway/Fermentation CrossRef 噪声

**问题**：`_CR_PATH_METHODS` 中单字 `"biosynthesis"` 和 `_CR_FERM_METHODS` 中宿主名
（`"Escherichia coli"` 等）单独作为 CrossRef 查询词，被模糊匹配到大量非 NNAA 文献（含多语言论文）。

**修复**：
- `_CR_PATH_METHODS`：`"biosynthesis"` 改为 `"amino acid biosynthesis"`，`"metabolic pathway"` 改为 `"metabolic pathway engineering"`
- `_CR_FERM_METHODS`：宿主名附加 `"amino acid"` 约束；移除单独的 `"Escherichia coli"` 等条目

### 7.3 PubMed Pathway 多语言噪声

**问题**：`_NNAA_GENERIC AND _PATH_SIG` 组合通过 PubMed 引入部分非英文期刊文献。

**修复**：该组合查询添加 `AND English[Language]` 限定。

### 7.4 AI 筛选 extra_instructions 重构（2026-07-20 二次迭代）

**问题（第一版）**：AI 要求标题/摘要中必须出现具名 NNAA 化合物，导致以通用术语（"non-natural amino acid synthesis"）描述的方法学论文误判为 false（假阴性）。

**问题（第二版）**：放宽后出现新的假阳性：
1. 天然标准氨基酸（Cys/Gly/Ser 等）的合成通路论文被纳入
2. 疾病/医学语境中以 NNAA 作为生物标志物的论文被纳入
3. 纯 GCE 应用论文（点击化学/蛋白标记）被纳入

**最终方案**（当前版本）：
- 核心原则：pass_filter=true 当且仅当文章核心贡献是报告 NNAA 的合成/制备/生产方法
- 明确新增排除规则：① 20 种标准氨基酸（即使方法新颖）② 疾病研究 ③ 应用研究 ④ 酶抑制/机制研究 ⑤ 非氨基酸产物 ⑥ 综述/应用综述 ⑦ 生物防御机制研究

**2023-05 测试验证结果**（PubMed 801 条 → AI 通过 25 条）：
- 精确率 = 22/25 = **88%**（超过 80% 目标）
- 假阴性率 = 极低（20篇拒绝样本中 0 篇为漏判）
- 批次错误 = 0（keep_on_batch_error=false 避免噪声混入）

## 6. 与 literature_fetch 的关系

本项目从 `literature_fetch`  fork 并精简，移除 BRENDA/数据引擎/补充库/酶学 notebook 策略等 NNAA 不需要的模块，保留 PubMed/CrossRef 抓取与 AI 筛选能力。
