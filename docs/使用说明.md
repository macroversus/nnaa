# 非天然氨基酸文献获取 — 使用说明

## 1. 项目简介

本项目在 `非天然氨基酸/` 目录下独立运行，用于获取**非天然氨基酸（NNAA）**相关文献，覆盖：

| 轨道 | 中文主题 | 典型方法 |
|------|----------|----------|
| `pathway` | 代谢通路 | 途径工程、遗传密码扩展、前体代谢 |
| `enzymatic` | 酶法合成 | 转氨酶、酶级联、无细胞翻译 |
| `fermentation` | 生物发酵 | 大肠杆菌/酵母细胞工厂、发酵产率 |
| `chemical` | 化学合成 | 不对称合成、偶联反应、全合成 |
| `hybrid` | 酶-化学联用 | chemoenzymatic、one-pot 多步 |

**标准流程**：检索 → AI 判断 → 关键结果输出

## 3. 环境准备

```bash
cd /root/非天然氨基酸
pip install -r requirements.txt
cp config.yaml.example config.yaml   # 编辑 email / API key
```

| 配置项 | 说明 |
|--------|------|
| `paths.master_csv` | 已有 DOI 主库（去重用） |
| `pubmed.email` | PubMed 必填真实邮箱 |
| `crossref.mailto` | CrossRef 必填真实邮箱 |
| `search.date_from` / `date_to` | 检索时间窗 |
| `ai_screening.enabled` | 是否启用 AI（完整流水线建议 `true`） |
| `ai_screening.api_key` 或环境变量 `NNAA_FETCH_OPENAI_API_KEY` | AI API 密钥 |

## 4. 独立入口：`nnaa_fetch.py`

推荐使用 **`nnaa_fetch.py`** 作为唯一入口，三步流水线一次完成：

```bash
# 完整流程：检索 → AI → 关键 CSV
python3 nnaa_fetch.py run

# 预览检索式（不请求 API）
python3 nnaa_fetch.py run --dry-run

# 查看内置关键词与检索式
python3 nnaa_fetch.py keywords
```

### 3.1 分步执行

```bash
# 阶段 1：仅检索 + 去重（结果写入 output/ 与 state/pending_ai.json）
python3 nnaa_fetch.py search

# 阶段 2：仅 AI 判断（需先完成 search，或存在 pending_ai.json）
python3 nnaa_fetch.py ai
python3 nnaa_fetch.py ai --resume          # 断点续跑

# 阶段 3：从最新 CSV 导出关键结果
python3 nnaa_fetch.py export
python3 nnaa_fetch.py export -i output/2026-06-25/all.csv
```

### 3.2 常用参数

```bash
python3 nnaa_fetch.py run \
  --date-from 2024-01-01 \
  --date-to 2026-06-25 \
  --track pathway              # 只跑代谢通路轨道
  --source pubmed              # 只跑 PubMed
  --force-ai                   # config 未启用 AI 时也强制跑 AI
  -v                           # 详细日志
```

## 5. 检索关键词来源

关键词集中在 **`nnaa_keywords.py`**（代码维护，非手工填 config）：

- **核心词**：non-natural / unnatural / noncanonical / non-proteinogenic amino acid、genetic code expansion 等
- **PubMed**：各轨道 6–7 条布尔检索式（Title/Abstract）
- **CrossRef**：核心词 × 方法信号词组合（每轨道约 40+ 条）

`config.yaml` 中 `pubmed.use_keywords_from_code: true`（默认）时优先使用代码词库；若在 config 中填写 `{track}_queries` 则覆盖代码默认值。

查看全部关键词：

```bash
python3 nnaa_fetch.py keywords
```

## 6. AI 判断说明

AI 模块（`ai_screener.py`）对每条文献输出：

| 字段 | 含义 |
|------|------|
| `ai_pass` | 是否通过筛选（非综述 + 主题相关 + 有实验） |
| `ai_nnaa_category` | pathway / enzymatic / fermentation / chemical / hybrid / both |
| `ai_synthesis_method` | 重点合成方法 |
| `ai_has_pathway_or_synthesis` | 是否明确报道通路或合成路线 |
| `ai_rationale_zh` | 中文简要理由 |

断点文件：`state/ai_checkpoint.json`（中断后用 `nnaa_fetch.py ai --resume` 续跑）

## 7. 关键结果输出

运行 `export` 或完整 `run` 后，在以下目录生成**关键文件**：

```
output/key_results/<时间标签>/
├── 关键_代谢通路.csv
├── 关键_酶法合成.csv
├── 关键_生物发酵.csv
├── 关键_化学合成.csv
├── 关键_酶化学联用.csv
├── 关键_全部可用文献.csv      ← 所有 ai_pass=true 的合并
└── 关键结果汇总.json          ← 数量统计与文件路径
```

**关键文献判定**：`ai_pass=true` 且非 API 错误占位行。

## 9. 故障排查

| 现象 | 处理 |
|------|------|
| PubMed 403 / 无结果 | 检查 `pubmed.email` 是否为真实邮箱 |
| CrossRef 超时 | 使用 `--source pubmed` 先跑 PubMed；或 `--resume-crossref`（底层 fetch_literature） |
| AI 跳过 | 设置 `NNAA_FETCH_OPENAI_API_KEY` 且 `ai_screening.enabled: true` |
| export 无文件 | 先完成 AI 阶段；或检查 `ai_pass` 列是否有 `true` |
| 主库路径不存在 | 修改 `paths.master_csv` 或暂时置空并关闭 `local_pdf` |

## 10. 与 literature_fetch 的关系

- **原型**：`literature_fetch`（酶/纳米酶）
- **本项目**：独立目录、独立入口 `nnaa_fetch.py`、独立关键词库 `nnaa_keywords.py`
- 底层复用：PubMed/CrossRef fetcher、去重、AI checkpoint 等模块
